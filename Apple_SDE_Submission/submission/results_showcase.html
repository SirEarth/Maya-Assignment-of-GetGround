⑦ Real Iteration Story · The "NZ 154-Rows" DQ Loop

Concrete example of Task C-2 closure: DQ → rule fix → replay → data promoted

Before resolver fix

154 rows
failing DQ_FMT_001 · resolver only accepted "New Zealand", rejected ISO code "NZ". All 154 rows trapped in dq_bad_records.

After resolver fix + replay

0 failures
extended _COUNTRY_NAME_MAP to accept ISO 3166-1 codes, replayed the same source_batch_id · all 154 rows promoted to fact_price_offer.
🔁 This is the system's value proposition. Without DQ + replay, those 154 rows would have either been silently dropped or polluted analytics. With the loop, business identifies the rule gap, fixes the dictionary, and recovers the data — no engineer needed for the rule iteration.
